Actúa como un equipo senior de ingeniería (backend, frontend, DevOps, UX) y genera una aplicación B2B de entregas a domicilio lista para producción.

## 🎯 OBJETIVO
Construir un SaaS web completo donde:
- Clientes crean pedidos
- Admin gestiona operación
- Sistema asigna repartidores por zona (manual y automático)
- Hay dashboard financiero real
- Incluye autenticación segura

## 🧱 STACK (OBLIGATORIO)
- Backend: Node.js + Express
- DB: PostgreSQL
- ORM: Prisma
- Auth: JWT + bcrypt
- Frontend: React (Vite) con UI moderna (Tailwind o similar)
- Estado: React Query o Context
- Validación: Zod o similar

## 🗂️ ESTRUCTURA
Monorepo:
- /server (API)
- /web (frontend)
- /prisma (schema)
- .env.example

## 🔐 AUTENTICACIÓN
- Registro y login
- Roles: ADMIN, CLIENTE, DRIVER
- Middleware de autorización por rol
- Persistencia de sesión (JWT en httpOnly cookie o header)

## 📦 MÓDULO DE PEDIDOS
Campos:
- pickup (string)
- delivery (string)
- zona (enum: Norte, Sur, Este, Oeste)
- pago (enum: Efectivo, Transferencia, Billetera)
- monto (number)
- status (enum: PENDIENTE, ASIGNADO, EN_RUTA, ENTREGADO)
- driverId (nullable)

Endpoints:
- POST /orders
- GET /orders (filtros por estado/fecha/cliente)
- PATCH /orders/:id (status, asignación)
- POST /orders/assign-auto (por zona)
- POST /orders/assign-manual (driverId)

## 🚚 REPARTIDORES (DRIVERS)
- CRUD de drivers
- Campo zona (puede cubrir varias)
- Disponibilidad (activo/inactivo)

## 🧭 ZONIFICACIÓN
- Tabla Zone
- Reglas simples de asignación por zona
- Función de auto-asignación: toma pedidos PENDIENTE y asigna driver disponible de la misma zona (round-robin o el de menor carga)

## 💰 FINANZAS
- Tabla Transaction (orderId, amount, type: ingreso/gasto, method)
- Endpoints:
  - GET /finance/summary?range=day|week|month
- KPIs:
  - ingresos, gastos, utilidad
  - pedidos por periodo
- Billetera digital por cliente (balance, movimientos)
  - POST /wallet/topup
  - GET /wallet
  - GET /wallet/transactions

## 📊 REPORTES
- Entregas por día/semana/mes
- Rendimiento por driver
- Filtros por fecha y zona

## 🗄️ PRISMA SCHEMA
Incluye modelos:
- User (id, email, password, role)
- Order (campos arriba)
- Driver (id, name, zonas[], active)
- Zone (id, name)
- Transaction (id, orderId, amount, type, method, createdAt)
- Wallet (userId, balance)
- WalletTx (id, userId, amount, type, createdAt)

## 🔌 API (Express)
- Rutas organizadas por módulo
- Middleware de auth y roles
